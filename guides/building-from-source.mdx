## Prerequisites

You need the following tools on your system before starting.

| Tool | Minimum version | Purpose |
|------|----------------|---------|
| Go | 1.24+ | Build the Sussurro binaries |
| C/C++ compiler | GCC or Clang | Compile whisper.cpp and llama.cpp |
| CMake | 3.15+ | Build system for C dependencies |
| Make | any | Run build targets |
| Git | any | Clone repositories |

### Install prerequisites

<Tabs>
  <Tab title="macOS">
    ```bash
    xcode-select --install   # Xcode Command Line Tools (provides clang, make, git)
    ```

    Then install Go from [go.dev/dl](https://go.dev/dl/).

    <Note>
      The macOS overlay uses Cocoa, QuartzCore, and CoreVideo — all included with Xcode Command Line Tools. No additional system packages are needed.
    </Note>
  </Tab>
  <Tab title="Arch / Manjaro">
    ```bash
    sudo pacman -S base-devel cmake git go
    ```
  </Tab>
  <Tab title="Ubuntu / Debian">
    ```bash
    sudo apt install build-essential cmake git golang-go
    ```
  </Tab>
  <Tab title="Fedora">
    ```bash
    sudo dnf install gcc gcc-c++ cmake git golang
    ```
  </Tab>
</Tabs>

## Build steps

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/cesp99/sussurro.git
    cd sussurro
    ```
  </Step>

  <Step title="Install UI build dependencies (Linux only)">
    The main `sussurro` binary requires GTK3, WebKit2GTK, and AppIndicator development headers. On macOS, no additional packages are needed.

    <Tabs>
      <Tab title="Arch / Manjaro">
        ```bash
        sudo pacman -S gtk3 webkit2gtk-4.1 libappindicator-gtk3

        # Optional: true wlr-layer-shell overlay on Wayland
        sudo pacman -S gtk-layer-shell
        ```
      </Tab>
      <Tab title="Ubuntu / Debian (22.04+)">
        ```bash
        sudo apt install libgtk-3-dev libwebkit2gtk-4.1-dev libayatana-appindicator3-dev

        # Optional: Wayland layer-shell overlay
        sudo apt install libgtk-layer-shell-dev
        ```
      </Tab>
      <Tab title="Fedora (38+)">
        ```bash
        sudo dnf install gtk3-devel webkit2gtk4.1-devel libappindicator-gtk3-devel
        ```
      </Tab>
    </Tabs>

    <Tip>
      `gtk-layer-shell` enables a true Wayland layer-shell overlay that stays above all windows. Without it, the overlay falls back to a floating window on Wayland and uses `_NET_WM_STATE_ABOVE` on X11. Install it if you are on Wayland.
    </Tip>

    <Note>
      If you only want to build `sussurro-transcribe` (the file transcription CLI), skip this step. The transcribe binary has no UI dependencies.
    </Note>
  </Step>

  <Step title="Build C/C++ dependencies">
    ```bash
    make deps
    ```

    This command:

    1. Clones `whisper.cpp` into `third_party/`
    2. Clones `go-llama.cpp` into `third_party/`
    3. Compiles static `.a` libraries — with Metal acceleration on macOS, CPU-optimized on Linux

    You only need to run `make deps` once, or again after updating the submodules.
  </Step>

  <Step title="Build the main binary">
    ```bash
    make build
    ```

    Produces `bin/sussurro` with the overlay, settings window, and system tray.

    `make build` also handles all platform-specific quirks automatically — see [How `make build` works internally](#how-make-build-works-internally) below.
  </Step>

  <Step title="Build the transcribe tool (optional)">
    ```bash
    make build-transcribe
    ```

    Produces `bin/sussurro-transcribe`. This target links only against Whisper and llama — no GTK, WebKit, or AppIndicator required.
  </Step>
</Steps>

## All make targets

| Target | Output | Description |
|--------|--------|-------------|
| `make deps` | `third_party/` | Clone and compile whisper.cpp and llama.cpp |
| `make build` | `bin/sussurro` | Main binary with overlay, settings window, and tray |
| `make build-transcribe` | `bin/sussurro-transcribe` | File transcription CLI (no UI dependencies) |
| `make run` | — | Build and immediately run the main binary |
| `make clean` | — | Remove `bin/` and compiled artifacts |

## How `make build` works internally

`make build` handles several platform quirks automatically. You do not need to do anything manually.

### webkit2gtk-4.0 compatibility shim (Arch Linux)

The settings window library (`webview_go`) hardcodes `pkg-config: webkit2gtk-4.0` in its CGO directives. Arch Linux ships `webkit2gtk-4.1` only.

`make build` auto-creates `.build-compat/pkgconfig/webkit2gtk-4.0.pc` — a shim `.pc` file that redirects pkg-config queries for `webkit2gtk-4.0` to the installed `webkit2gtk-4.1`. It then sets `PKG_CONFIG_PATH` to include this directory for the duration of the build. No manual steps needed.

### AppIndicator variant detection

The system tray library supports two AppIndicator backends:

| Backend | Library | Distros |
|---------|---------|---------|
| Default (ayatana) | `ayatana-appindicator3-0.1` | Ubuntu, Fedora, openSUSE |
| Legacy | `appindicator3-0.1` | Arch Linux, Manjaro |

`make build` probes `pkg-config` for both libraries and automatically adds `-tags legacy_appindicator` when only the Arch variant is found.

### gtk-layer-shell detection

`make build` checks for `gtk-layer-shell` via `pkg-config`. If found, it compiles the overlay with true wlr-layer-shell support (proper Wayland overlay, always above all windows). If not found, the overlay falls back to a regular floating window.

## First run

```bash
./bin/sussurro
```

<Warning>
  Always launch Sussurro from a terminal. Launching via a desktop icon or application menu is not yet supported — the overlay and tray will not work correctly outside a terminal session.
</Warning>

On first run, Sussurro creates `~/.sussurro/config.yaml` and prompts you to download the required AI models into `~/.sussurro/models/`:

| Model | Size | Purpose |
|-------|------|---------|
| Whisper Small | ~488 MB | Faster ASR, good accuracy |
| Whisper Large v3 Turbo | ~1.62 GB | Slower ASR, best accuracy |
| Qwen 3 Sussurro LLM | ~1.28 GB | Text cleanup (always required) |

You can also place model files manually and update their paths in `~/.sussurro/config.yaml`.

To run in headless mode (no overlay or tray):

```bash
./bin/sussurro --no-ui
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="pkg-config: webkit2gtk-4.0 not found">
    This happens if you run `go build` directly instead of `make build`. The `make build` target creates the webkit2gtk-4.0 compatibility shim automatically. Always use `make build`.
  </Accordion>
  <Accordion title="appindicator3-0.1 not found">
    The wrong AppIndicator library is being selected. Use `make build`, which auto-detects the correct variant (`ayatana` or `legacy`) for your system.
  </Accordion>
  <Accordion title="gtk-layer-shell: not found (warning, not error)">
    This is a warning, not a build failure. The overlay will use a regular floating window instead of a true Wayland layer. To get the full Wayland overlay, install `gtk-layer-shell` and rebuild:

    ```bash
    # Arch
    sudo pacman -S gtk-layer-shell

    # Ubuntu / Debian
    sudo apt install libgtk-layer-shell-dev
    ```

    Then run `make build` again.
  </Accordion>
  <Accordion title="macOS: xcode-select: error">
    Run `xcode-select --install` and accept the license agreement. If the tools are already installed but the command still fails, reset the active developer directory:

    ```bash
    sudo xcode-select --reset
    ```
  </Accordion>
  <Accordion title="fatal error: gtk/gtk.h: No such file or directory">
    GTK3 development headers are missing. Install them for your distro:

    ```bash
    # Ubuntu / Debian
    sudo apt install libgtk-3-dev

    # Arch
    sudo pacman -S gtk3
    ```

    Then run `make build` again.
  </Accordion>
</AccordionGroup>
