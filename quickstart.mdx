## Install Sussurro

Run the one-line installer. It detects your platform, downloads the right binary, and places it in `/usr/local/bin` (or `~/.local/bin` if you don't have sudo):

```bash
curl -fsSL https://raw.githubusercontent.com/cesp99/sussurro/master/scripts/install.sh | bash
```

Supports Linux (X11 and Wayland) and macOS.

<Note>
**Linux users:** you may need to install a few system libraries first. See [Installation](/installation) for per-distro commands. **macOS users:** no extra dependencies are needed.
</Note>

## First run

Launch Sussurro from a terminal:

```bash
sussurro
```

<Warning>
Always launch from a terminal — not by double-clicking the binary or using a `.desktop` shortcut. The overlay and tray will not appear correctly otherwise.
</Warning>

On first run, Sussurro guides you through downloading the AI models:

| Model | Size | Purpose |
|-------|------|---------|
| Whisper Small | ~488 MB | Fast speech recognition |
| Whisper Large v3 Turbo | ~1.62 GB | Higher accuracy speech recognition |
| Qwen 3 Sussurro LLM | ~1.28 GB | Text cleanup (always required) |

Choose a Whisper model (Small is recommended for most users). The Qwen 3 LLM is always downloaded. After download completes, the overlay capsule appears at the bottom of your screen.

## Use the hotkey

<Steps>
  <Step title="Focus any text field">
    Click inside any application where you want text to appear — a browser, editor, chat app, anything.
  </Step>
  <Step title="Hold the hotkey">
    **Linux X11 / macOS:** Hold `Ctrl+Shift+Space` (Linux) or `Cmd+Shift+Space` (macOS). The capsule animates to show recording is active.

    **Linux Wayland:** See [Wayland Setup](/platform/wayland) — requires a one-time shortcut configuration in your desktop environment.
  </Step>
  <Step title="Speak">
    Speak naturally. Filler words, self-corrections, and stutters are handled automatically.
  </Step>
  <Step title="Release">
    Release the hotkey. The capsule shows "transcribing", then your cleaned text appears in the focused application.
  </Step>
</Steps>

## Access Settings

Open the Settings window to switch Whisper models, change the transcription language, or reconfigure the hotkey:

<CardGroup cols={2}>
  <Card title="Via system tray" icon="circle-dot">
    Click the Sussurro icon in your system tray → **Open Settings**
  </Card>
  <Card title="Via overlay capsule" icon="computer-mouse">
    Right-click the pill-shaped capsule at the bottom of your screen → **Open Settings**
  </Card>
</CardGroup>

All changes take effect immediately — no restart required.

## macOS: grant Accessibility permission

On macOS, Sussurro needs Accessibility access to register a global hotkey:

<Steps>
  <Step title="Open System Settings">
    Go to **System Settings → Privacy & Security → Accessibility**
  </Step>
  <Step title="Add Sussurro">
    Click the **+** button and add the Sussurro binary
  </Step>
  <Step title="Enable the toggle">
    Make sure the toggle next to Sussurro is **on**
  </Step>
  <Step title="Relaunch">
    Quit and relaunch Sussurro from the terminal
  </Step>
</Steps>

<Tip>
If the hotkey still doesn't work after granting permission, toggle Accessibility off and back on, then relaunch.
</Tip>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Overlay doesn't appear">
    On Linux, verify GTK3 is installed: `pkg-config --exists gtk+-3.0 && echo ok`. See [Dependencies](/reference/dependencies) for install commands.
  </Accordion>
  <Accordion title="No text appears after speaking">
    Speak for at least 2 seconds. Run with `--no-ui` to see terminal output for errors. Check that your microphone is working.
  </Accordion>
  <Accordion title="Hotkey doesn't work (macOS)">
    Check Accessibility permission in **System Settings → Privacy & Security → Accessibility**. Toggle it off and on, then relaunch Sussurro.
  </Accordion>
  <Accordion title="Hotkey doesn't work (Wayland)">
    Wayland requires a one-time external shortcut setup. See the [Wayland Setup](/platform/wayland) guide.
  </Accordion>
  <Accordion title="'clipboard failed' error">
    On Wayland, install `wl-clipboard`: `sudo apt install wl-clipboard` (Ubuntu) or `sudo pacman -S wl-clipboard` (Arch).
  </Accordion>
</AccordionGroup>
